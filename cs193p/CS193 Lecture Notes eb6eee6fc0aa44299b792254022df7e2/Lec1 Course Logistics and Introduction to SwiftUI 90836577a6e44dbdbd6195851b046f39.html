Looking in to the Hello World App

struct ContentView: View {
	var body: some View {
		Text("Hello World!")
	}
]

struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        ContentView()
    }
}

The Keyword some

  • var body: some View where some View is type. some keyword means that it can be any type as long as it behaves like View
  • some View could be Text("Hello World!") in this example. some View is better format because the compiler makes the decision. (To choose from the subclasses of View)
  • If some View is not used, changes of the return type changes which would also require changing this.

One-line function

  • One line function doesn’t need return keyword.
  • It’s normal to use labels when giving parameters to functions. return RoudedRectangle(cornerRadius: 10.0)
  • If the last argument of a function is curly brace function it can be written as abc(first: 1) {stuff} after the function call. If no other arguments, () can be removed too.

An example of setting up a ContentView using ZStack

struct ContentView: View {
	var body: some View {
    return ForEach(0..<4, content: { index in
      ZStack(content: {
			RoundedRectangle(cornerRadius: 10.0)
				.fill(Color.white)
        	RoundedRectangle(cornerRadius: 10.0)
				.stroke(lineWidth: 3)
        	Text(“👻”)
      })
    })
		.padding()	// or .padding(10)
		.foregroundColor(Color.orange)	// or (.orange)
      .font(Font.largeTitle)	// or (.largeTitle)
  }
}
  • 0..<4 is same as [0, 1, 2, 3]. 0...4 means [0, 1, 2, 3, 4]

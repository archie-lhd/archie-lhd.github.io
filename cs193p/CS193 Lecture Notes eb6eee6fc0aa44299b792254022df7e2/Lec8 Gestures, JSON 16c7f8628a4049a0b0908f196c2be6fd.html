    UserDefaults

    "Persistent dictionary"

    Should only be used for "User preferences" or other lightweight bits of information

    Property Lists

    UserDefaults can only store what is called a Property List.

    This is NOT a protocol or a struct or anything tangible or Swift-like

    Property List is simply a "concept":

    • Any combination of String, Int, Bool, floating point, Data, Array, or Dictionary
    • If you want to store anything else, convert it into a Property List
    • Use Codable protocol in Swift: it converts structs into Data objects (where Data is a Property List)

    The "Any" Type

    The API for UserDefaults contain a lot of uses of the type Any, which is strange because it is pre-Swift.

    Since Swift is a strongly-typed language, it does not like Any yet supports it for backwards compatibility.

    Using UserDefaults

    To store data ...

    let defaults = UserDefaults.standard

// object must be a Property List
defaults.set(object, forKey: "SomeKey")
defaults.setDouble(37.5, forKey: "MyDouble")

    To retrieve data ...

    let i: Int = defaults.integer(forKey: "MyInteger")
let b: Data = defaults.data(forKey: "MyData")
let u: URL = defaults.url(forKey: "MyURL")
let strings: [String] = defaults.stringArray(forKey: "MyString")

    Non-Discrete Gestrues

    You also handle the gesture while it is in the process of happening (fingers are moving).

    Examples: DragGesture, MagnificationGesture, RotationGesture.

    LongPressGesture can also be treated as non-discrete (i.e. fingers down and fingers up).

    Handling Non-Discrete Gestures

    var theGesture: some Gesture {
	DragGesture(...)
		.onEnded { value in /* do something */ }
}

    What that value is varies from gesture to gesture ...

    • For a DragGesture, it‚Äôs a struct with things like the start and end location of the fingers.
    • For a MagnificationGesture it‚Äôs the scale of the magnification (how far the fingers spread out).
    • For a RotationGesture it‚Äôs the Angle of the rotation (like the fingers were turning a dial).

    The only place you can change your @GestureState ...

    @GestureState var myGestureState: MyGestureStateType = <starting value>
var theGesture: some Gesture {
	DragGesture(...)
		.updating($myGestureState) { value, myGestureState, transaction in
			myGestureState = /* usually something related to value */
	}
	.onEnded { value in /* do something */ }
}
    • This .updating will cause the closure you pass to it to be called when the fingers move.
    • The value argument to your closure is the same as with .onEnded (the state of the fingers).
    • We‚Äôre going to ignore the transaction argument (it has to do with animation).

    Tracking the absolute (actual) finger positions ...

    var theGesture: some Gesture {
	DragGesture(...)
		.onChanged { value in
			/* do something with value (the state of the fingers) */
		}
		.onEnded { value in /* do something */ }
}
